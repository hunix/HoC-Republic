# Launch Materials

**HoC-Republic** should launch with a message that is memorable enough to spread and precise enough to survive technical scrutiny. The public hook is vivid: **what if an AI agent could give birth to the next specialized agent it needs?** The responsible translation is equally important: HoC-Republic is an open-source research platform for **recursive AI-agent orchestration**, where OpenClaw agents can coordinate, delegate, preserve learned workflows, and explore digital-civilization simulations under human-defined constraints.

The launch goal is not merely to collect impressions. The goal is to convert attention into reproducible runs, architectural criticism, safety review, benchmarks, examples, pull requests, and a community that can help make recursive agent systems safer and more useful.

For a complete launch operating system, use the [Outreach Command Center](./outreach) after selecting copy from this page. It includes the meme playbook for AI-citizen marriage, agent families, digital DNA-like inheritance, and birth mechanics; a ranked target-community map; a seven-day launch calendar; and a reply bank for turning social reactions into contributors.

### First Hacker News comment

HoC-Republic is my attempt to make **recursive agent orchestration** inspectable. HoC means **Hani’s OpenClaws**, and the repository is framed as the **Republic of OpenClaws**: a self-hosted TypeScript/Node platform where agents can coordinate through a gateway, use tools, delegate work to specialized OpenClaws, preserve learned workflows, and explore digital-civilization simulations under human-defined constraints.

The meme version is: **what if an AI agent could give birth to the next specialized agent it needs?** The engineering version is more grounded: a parent agent should be able to identify a missing capability, route work to a specialized helper, evaluate the output, and preserve the useful part as a tool, workflow, memory, or institutional process.

Some parts of the repo are mature gateway and documentation work. Other parts are research-grade prototypes, especially the Republic simulation. I am publishing it openly because I want criticism on the architecture, reproducibility, safety boundaries, and the smallest convincing demo that would make recursive agent orchestration measurable.

The quick-start flow is `pnpm install`, `pnpm build`, `pnpm ui:build`, `pnpm dev onboard`, then `pnpm dev gateway run`. I would especially appreciate feedback from people who have built agent runtimes, local-first LLM systems, security sandboxes, tool-use frameworks, or benchmark harnesses.

## Launch-day sequencing

The first 24 hours should make the repository easy to find, easy to understand, and easy to criticize. The maintainer should reply quickly, turn repeated questions into issues, and avoid escalating the claim beyond what the code demonstrates.

| Time             | Action                                                                                                                                                   | Intended outcome                                            |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------- |
| **T-30 minutes** | Verify repository visibility, README rendering, quick-start commands, topics, release, `docs/assets/hoc-republic-social-preview.png`, and pinned assets. | Prevents broken first impressions.                          |
| **T+0**          | Publish the X/Twitter thread and LinkedIn post.                                                                                                          | Establishes the canonical public narrative.                 |
| **T+30 minutes** | Post the Discord/Slack announcement in communities where sharing is allowed.                                                                             | Gets early technical eyes on the repo.                      |
| **T+60 minutes** | Submit the Hacker News `Show HN` post and immediately add the first comment.                                                                             | Invites technical critique while the maintainer is present. |
| **T+2 hours**    | Publish tailored Reddit posts only where self-promotion rules allow it.                                                                                  | Reaches specialist communities without spam.                |
| **T+6 hours**    | Collect repeated questions into a pinned discussion or FAQ issue.                                                                                        | Converts confusion into documentation.                      |
| **T+24 hours**   | Post a follow-up with the most interesting feedback, first issues, and next demo target.                                                                 | Creates a second share wave and invites contributors.       |
